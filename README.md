# Diggi Finmodel

Це статична HTML-версія фінансової моделі Diggi для публікації через GitHub Pages.

## Що всередині

- `index.html` - основна сторінка фінмоделі.
- `.nojekyll` - службовий файл для GitHub Pages, щоб сайт публікувався як звичайний статичний HTML.

## Як опублікувати через GitHub Pages

1. Створи новий репозиторій на GitHub, наприклад `diggi-finmodel`.
2. Завантаж у репозиторій усі файли з цієї папки.
3. Відкрий у репозиторії `Settings`.
4. Перейди в `Pages`.
5. У блоці `Build and deployment` вибери:
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`
   - `Folder`: `/root`
6. Натисни `Save`.
7. Через 1-3 хвилини GitHub покаже посилання на сайт.

Посилання буде приблизно такого формату:

```text
https://your-github-name.github.io/diggi-finmodel/
```

## Як оновлювати модель

Коли файл `index.html` змінюється, потрібно зробити новий commit у GitHub. Після цього GitHub Pages автоматично опублікує нову версію.

## Важливе обмеження

GitHub Pages публікує сайт, але не є спільною базою даних.

Поточна модель зберігає зміни в браузері користувача. Це означає, що якщо кілька керівників відкриють посилання, кожен може мати свої локальні дані.

Щоб керівники одночасно працювали з одними й тими самими даними, наступним етапом треба підключити спільну базу, наприклад Firebase або Supabase.
